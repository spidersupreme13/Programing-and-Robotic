Programacion

La programacion(en el sentido de la robotica) es tecnicamente darle intrucciones y/o que haceres a una maquina. Esto nos sirve para enseñarle a un robot a saludar, enseñarle a un servo motor en que angulo girar, enseñarle a una luz RGB en que momento cambir de color, etc.

Como funcionan los codigos(en arduino)

  • Void setup(){ texto }

    • El void setup es para dar la orden de que algo ocurra una sola vez y no vuelva a ocurrir

  • Void loop(){ texto }

    • El void loop hace que una accion ocurra infinitamente

  • PinMode(pin, modo);

    • El pinMode es para decirle al arduino que tipo de pin es el que estamos usando, ya sea digital o analogico

  • DigitalWrite(pin, estado);

    • El digitalWrite es para decirle al arduino que pin va a ser usado y que estado va a tener, ya sea HIGH o LOW

  • AnalogRead(pin);

    • El analogRead es para leer un pin analogico y obtener un valor entre 0 y 1023

  • Delay(tiempo);

    • El delay es para darle un tiempo de espera a una accion, ya sea 1000 milisegundos o 5000 milisegundos

  • AnalogWrite(pin, valor);

    • El analogWrite es para escribir un valor entre 0 y 255 en un pin analogico

  • Interrupts(pin, funcion, modo);

    • El interrupts es para activar una interrupcion en un pin, ya sea para detectar un cambio de estado o un flanco de subida o bajada

  • AttachInterrupt(pin, funcion, modo);

    • El attachInterrupt es para adjuntar una interrupcion a un pin, ya sea para detectar un cambio de estado o un flanco de subida o bajada

  • DetachInterrupt(pin);

    • El detachInterrupt es para desactivar una interrupcion en un pin, ya sea para detectar un cambio de estado o un flanco de subida o bajada

  • PulseIn(pin, estado);

    • El pulseIn es para medir el tiempo que dura un pulso en un pin, ya sea en estado HIGH o LOW

  • PulseInLong(pin, estado);

    • El pulseInLong es para medir el tiempo que dura un pulso en un pin, ya sea en estado HIGH o LOW, pero con una mayor precision

  • DelayMicroseconds(tiempo);

    • El delayMicroseconds es para darle un tiempo de espera en microsegundos a una accion, por ejemplo, 1000 microsegundos

  • Millis();

    • El millis es para obtener el tiempo transcurrido desde que se inicio el programa en milisegundos

  • Micros();

    • El micros es para obtener el tiempo transcurrido desde que se inicio el programa en microsegundos

  • Second();

    • El second es para obtener el segundo actual del reloj interno del arduino

  • Time();

    • El time es para obtener la hora actual del reloj interno del arduino en formato de hora, minuto y segundo

  • Date();

    • El date es para obtener la fecha actual del reloj interno del arduino en formato de dia, mes y año

  • TimeDate();

    • El timeDate es para obtener la fecha y hora actual del reloj interno del arduino en formato de dia, mes, año, hora, minuto y segundo

  • Map(valor, minimo, maximo, nuevoMinimo, nuevoMaximo);

    • El map es para mapear un valor de un rango a otro rango, por ejemplo, de 0 a 1023 a 0 a 255

  • Constrain(valor, minimo, maximo);

    • El constrain es para restringir un valor a un rango determinado, por ejemplo, de 0 a 255

  • Random(minimo, maximo);

    • El random es para generar un numero aleatorio entre un minimo y un maximo, por ejemplo, entre 0 y 100

  • RandomSeed(seed);

    • El randomSeed es para inicializar el generador de numeros aleatorios con una semilla, por ejemplo, con el valor de millis()

  • RandomRange(minimo, maximo);

    • El randomRange es para generar un numero aleatorio entre un minimo y un maximo, pero con una mayor precision

  • RandomFloat(minimo, maximo);

    • El randomFloat es para generar un numero decimal aleatorio entre un minimo y un maximo, por ejemplo, entre 0.0 y 1.0

  • RandomInt(minimo, maximo);

    • El randomInt es para generar un numero entero aleatorio entre un minimo y un maximo, por ejemplo, entre 0 y 100

  • Serial.begin(baudrate);

    • El Serial.begin es para iniciar la comunicacion serial con el arduino, ya sea a 9600 o 115200 baudrate(velocidad de transmision de datos)

  • Serial.print(texto);

    • El Serial.print es para imprimir un texto en el monitor serial del arduino

  • Serial.println(texto);

    • El Serial.println es para imprimir un texto en el monitor serial del arduino y hacer un salto de linea

  • Serial.read();

    • El Serial.read es para leer un dato del monitor serial del arduino

  • Serial.available();

    • El Serial.available es para saber si hay datos disponibles en el monitor serial del arduino

  • Serial.flush();

    • El Serial.flush es para limpiar el buffer del monitor serial del arduino

  • Serial.write(dato);

    • El Serial.write es para escribir un dato en el monitor serial del arduino

  • Serial.readString();

    • El Serial.readString es para leer una cadena de texto del monitor serial del arduino

  • Serial.readBytes(buffer, longitud);

    • El Serial.readBytes es para leer un numero determinado de bytes del monitor serial del arduino y guardarlos en un buffer

  • Serial.readBytesUntil(delimitador, buffer, longitud);

    • El Serial.readBytesUntil es para leer bytes del monitor serial del arduino hasta que se encuentre un delimitador y guardarlos en un buffer

  • Serial.parseInt();

    • El Serial.parseInt es para leer un numero entero del monitor serial del arduino

  • Serial.parseFloat();

    • El Serial.parseFloat es para leer un numero decimal del monitor serial del arduino

  • Serial.peek();

    • El Serial.peek es para leer el siguiente byte del monitor serial del arduino sin eliminarlo del buffer

  • Serial.readStringUntil(delimitador);

    • El Serial.readStringUntil es para leer una cadena de texto del monitor serial del arduino hasta que se encuentre un delimitador

  • Serial.end();

    • El Serial.end es para finalizar la comunicacion serial con el arduino
